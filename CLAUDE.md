# CLAUDE.md

## Project overview
Dokime (dokimi, Greek: "the test of quality") is an open-source data curation toolkit for ML training datasets. Positioning: "pytest for your training data." It provides a CLI, Python SDK, and web UI for scoring, filtering, deduplicating, and exploring data.

## Architecture
```
src/dokime/
  __init__.py          -- package root, version re-export
  _version.py          -- auto-generated by setuptools-scm
  cli.py               -- Typer-based CLI (10 commands: version, curate, stats, embed, search, outliers, score, push, attribute, explore)

  core/
    filters.py         -- Base Filter ABC + 12 built-in heuristic filters
    pipeline.py         -- Pipeline orchestration (YAML config, per-filter stats, streaming execution, per-doc error handling)
    registry.py         -- Filter registry for YAML-based pipeline construction + custom filter registration

  io/
    readers.py          -- auto_read() + readers for JSONL, Parquet (streaming via iter_batches), CSV, HuggingFace datasets
    writers.py          -- write_jsonl(), write_parquet(), StreamingWriter (batched flush)
    hub.py              -- HuggingFace Hub integration (load_from_hub, push_to_hub)

  quality/
    scoring.py          -- QualityScorer (9 signals + composite), TokenCountFilter, PerplexityFilter
    dedup.py            -- ExactDedup (SHA-256), MinHashDedup (datasketch LSH)
    report.py           -- run_report() — the `dokime score` terminal report (grade A-F, issues, distribution, worst docs)

  embeddings/
    compute.py          -- EmbeddingModel wrapper (sentence-transformers), compute_embeddings() with missing-field warnings
    search.py           -- EmbeddingIndex (FAISS), AnomalyScorer (k-NN outlier detection)
    dedup.py            -- Semantic dedup: find_semantic_duplicates(), deduplicate_by_embeddings()

  explore/
    server.py           -- FastAPI web UI server (browse, search, stats, UMAP visualization)
    static/index.html   -- Single-page HTML frontend

  attribution/
    engine.py           -- TRAK-based data attribution
    model_output.py     -- Model output wrapper

examples/
  basic_pipeline.yaml      -- Minimal YAML config example
  advanced_pipeline.yaml   -- Comprehensive YAML config with all filters
  quickstart.py            -- Runnable script: load, filter, dedup, embed, search, outliers

tests/                     -- pytest test suite
```

## All available filters (in registry)
1. `LengthFilter` -- min/max character length
2. `WordCountFilter` -- min/max word count
3. `LineCountFilter` -- min/max line count
4. `WhitespaceFilter` -- max whitespace ratio
5. `RepetitionFilter` -- max n-gram repetition ratio
6. `SpecialCharFilter` -- max special character ratio
7. `AlphaFilter` -- min alphabetic character ratio
8. `URLFilter` -- max URL character ratio
9. `StopwordFilter` -- min stopword ratio (catches keyword spam)
10. `LanguageFilter` -- language detection (lingua or fastText backend) [requires `[nlp]`]
11. `FieldExistsFilter` -- require a non-empty field
12. `RegexFilter` -- include/exclude by regex pattern
13. `ExactDedup` -- SHA-256 exact dedup
14. `MinHashDedup` -- MinHash-LSH fuzzy dedup [requires `[dedup]`]

## CLI commands
```
dokime version       -- print version
dokime curate        -- run curation pipeline (flags or --config YAML)
dokime stats         -- show dataset statistics (supports --field)
dokime embed         -- compute embeddings (sentence-transformers), save as .npy
dokime search        -- semantic search over a dataset (supports --field)
dokime outliers      -- find anomalous documents via k-NN distance (supports --field)
dokime score         -- grade dataset quality A-F with actionable report (supports --field)
dokime push          -- push dataset to HuggingFace Hub
dokime attribute     -- TRAK-based data attribution (supports --field)
dokime explore       -- launch web UI
```

## Key conventions
- Python 3.10+, type hints on all public APIs
- src/ layout, single package `dokime`
- Ruff for linting and formatting (line-length 119)
- All filters inherit from `dokime.core.filters.Filter` (ABC with `filter()` and `name()`)
- Optional dependencies via extras: `[io]`, `[nlp]`, `[embeddings]`, `[dedup]`, `[explore]`, `[attribution]`, `[all]`
- Lazy imports for optional dependencies with clear ImportError messages
- Filters are registered in `dokime.core.registry.FILTER_REGISTRY` for YAML config support
- Custom filters can be registered via `register_filter(name, cls)`
- Pipeline handles per-document errors gracefully (logs warning, continues processing)
- All CLI commands with text output respect the `--field` parameter for custom field names

## Testing
- `pytest` with markers: `@pytest.mark.slow`, `@pytest.mark.gpu`
- Run fast tests: `pytest -m "not slow"`
- Tests live in `tests/` mirroring `src/dokime/` structure
